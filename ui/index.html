Quick start

VoiceQA helps you QA what was said and how it sounded.

Run a single clip, or run curated eval suites with local audio and review the exact flagged moments.
Suggested demos
  • symptom-triage — “should pass” baseline suite (terms + vitals)
  • hallucination-demo — intentional additions/omissions/contradictions
  • prosody-demo — deterministic weird pauses / pacing (flags + jump-to)
Tips
  • Audio is local-only; manifests are committed.
  • Use Save Baseline + Compare to track regressions.
  • Use the per-case Jump buttons to listen to flagged regions.

Analyse one audio file

Upload an audio file and the expected script. VoiceQA will transcribe, score, and return verdict + metrics.

Tip: if Ollama is not running, VoiceQA will still complete end-to-end and generate a deterministic report.

Run an eval suite

Suites live in eval_set/suites/. Manifests are committed; audio files are usually local-only.

Suite manifests can include expected_terms for hard symptom/medication words (e.g. dyspnea, hemoptysis, levetiracetam). These are checked via term_fidelity.
If audio files are missing, cases will FAIL fast with a “Missing audio file” reason.
